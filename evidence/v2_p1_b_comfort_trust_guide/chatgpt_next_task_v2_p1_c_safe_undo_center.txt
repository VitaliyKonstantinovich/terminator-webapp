Почему именно сейчас

У вас уже есть:

Guardian / Safe Mode

approvals

rollback

диагностика

comfort/trust guide

Но обычный пользователь по-прежнему задает один вопрос:

"А если Мина ошибется, смогу ли я все вернуть?"

Премиальность появляется не тогда, когда система редко ошибается.

Она появляется тогда, когда пользователь не боится ошибок.

Поэтому следующий маленький production-блок:

Сделать существующие rollback-механизмы человеческими.

Цель

Добавить визуальный центр восстановления последних действий.

Не новый rollback-engine.

Только UX-слой поверх уже существующих событий и снапшотов.

Scope (маленький)

Новый блок в System Integration рядом с "Проводником Мины".

Название:

Безопасное восстановление

Показывать последние 5–10 важных действий:

создана задача

обновлена память

изменен проект

импортированы данные

выполнена автоматизация

Для каждого:

время

краткое описание

статус

доступность восстановления

UI

Карточка:

2 минуты назад

Создан план проекта

Безопасность:
изменение можно отменить

[Подробнее]
[Восстановить]
Важный UX-текст

Не использовать:

Rollback
Snapshot
Event log

Использовать:

Вернуть предыдущее состояние
Восстановить
Отменить изменение

Это значительно лучше работает для нетехнических пользователей.

Дополнительная карточка

Наверху блока:

Если что-то пошло не так,
Мина может помочь вернуть
последние безопасные изменения.

Это снижает тревожность.

Что НЕ делать

Не делать:

полноценную timeline

diff viewer

event sourcing

историю на тысячи событий

восстановление памяти агента

Это уже следующий этап.

Архитектура

Добавить:

buildRecoverySnapshot()

возвращающий:

JavaScript
{
  recentActions: [],
  recoverableCount: 0,
  lastRecoverableAt: ""
}

Только чтение существующих данных.

Никакой новой бизнес-логики.

Acceptance Criteria
AC-1

Если есть recoverable action:

Восстановить

кнопка отображается.

AC-2

Если действие нельзя вернуть:

Не требует восстановления

или

Восстановление недоступно
AC-3

Пользователь понимает:

что изменилось;

можно ли вернуть;

насколько это безопасно.

Без открытия DevTools и логов.

AC-4

Последние действия читаются на мобильном экране 390 px без горизонтального скролла.

AC-5

Экспертные детали скрыты в <details>.

По умолчанию показывается только человеческое описание.
