# Terminator V2 11/10 Roadmap

Status: PASS

V2 starts from the accepted V1 Productized RC. It does not rewrite V1. It hardens the product line toward production V2 final.

## Phase V2-P0

Goal: make the product safe, understandable, recoverable and reliable.

1. First Run 11/10
   - Owner Command Center.
   - Setup Route.
   - Scheme Mina readiness explanation.
   - Owner-assisted checklist clarity.
   - One next action.

2. Recovery 11/10
   - Diagnost.
   - Codex Repair.
   - Recovery Wizard.
   - Rollback.
   - Clear repair package.

3. Memory Search 11/10
   - Strong relevance.
   - Large dataset readiness.
   - Task/file/artifact/evidence refs.
   - Index freshness and rebuild UX.

4. Safety / Control 11/10
   - Guardian.
   - Approval.
   - Cost Guard.
   - Privacy Guard.
   - Emergency Stop.

## Phase V2-P1

Goal: turn product quality into repeatable operations.

1. QA Autotest Factory
   - Stable product scenarios.
   - Fixture-based acceptance.
   - Regression matrix.
   - Owner-readable test results.

2. ResearchOps + Brain Council polish
   - Better Council profiles.
   - Better source gate.
   - Better Decision Passport.
   - Stronger contradiction handling.

3. Product Shell
   - Windows companion/tray.
   - Autostart status.
   - Safe restart.
   - Local logs.
   - No intrusive windows.

## Phase V2-P2

Goal: deepen capabilities after core product reliability is stable.

1. Advanced Eyes
   - Visual evidence.
   - Visual diff.
   - Mobile visual smoke.
   - Evidence binding.

2. Advanced Hands / Adapters
   - Safe file/code/system adapters.
   - Adapter permission matrix.
   - Guardian/Verifier/Rollback enforced.

3. Advanced Voice
   - Push-to-talk.
   - Intent preview.
   - Confirmation.
   - Voice safety gates.

## Phase V2-P3

Goal: extend beyond owner-first product after the core is mature.

1. Marketplace / Plugins
   - Plugin registry.
   - Safety constraints.
   - Install/disable model.

2. Premium Visual Layer
   - Bespoke Mina asset.
   - Final visual polish.
   - Premium interaction details.

3. Team / Enterprise
   - Roles.
   - Permissions.
   - Shared workflows.
